Enable outbound delivery channels

Add a shared outbound tool path for proactive user-facing updates.
This lets agent flows deliver messages to the web dashboard and
Telegram when a direct inline response is not enough.

The dashboard now stores outbound messages in a bounded queue,
exposes them through a polling API, and shows them in the UI.

AgentCore now exposes send_outbound_message and sends Telegram
messages from the configured bot settings with soft failure handling.
