Refactor build system to make crun and containers optional

Added `--with-crun` option to deployment and build scripts.
By default, `deploy.sh` and CMake will now build TizenClaw
in standard Native Debug mode without building the OCI runtime
(`crun`) or packing the container rootfs artifacts, significantly
speeding up iterative development. In this default mode, all 
services including `tizenclaw` and `tizenclaw-code-sandbox` 
run natively outside of containers directly on the host system.
The fully containerized structure is now strictly opt-in via
`deploy.sh --with-crun`. Updated `README.md` to reflect these changes.
